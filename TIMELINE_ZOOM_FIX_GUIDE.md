# Руководство по исправлению наложения меток времени при масштабировании таймлайна

## Проблема

При масштабировании таймлайна (уменьшении масштаба) метки времени накладывались друг на друга, что делало их нечитаемыми.

## Решение

Был реализован адаптивный алгоритм расстановки меток времени, который автоматически регулирует интервал между метками в зависимости от текущего масштаба.

## Как это работает

### 1. Динамический интервал меток

- **Нормальный масштаб**: Интервал 5 секунд (как и раньше)
- **Малый масштаб**: Интервал автоматически увеличивается для предотвращения наложения

### 2. Расчет интервала

```python
min_spacing_px = 80  # Минимальное расстояние между метками в пикселях
current_spacing_px = pixels_per_frame * fps * 5  # Текущее расстояние при 5-секундном интервале

if current_spacing_px < min_spacing_px:
    # При малом масштабе увеличиваем интервал
    step_seconds = max(5, int(min_spacing_px / (pixels_per_frame * fps)))
```

### 3. Округление до удобных значений

Интервал автоматически округляется до удобных значений:
- 5-7 секунд → 5 секунд
- 8-12 секунд → 10 секунд  
- 13-20 секунд → 15 секунд
- 21-40 секунд → 30 секунд
- 41+ секунд → 60 секунд

### 4. Центрирование текста

Текст меток времени теперь центрируется относительно засечки, что улучшает визуальное восприятие.

### 5. Проверка перекрытия

Перед отрисовкой каждой метки проверяется, не перекрывается ли она с предыдущей:
```python
if text_x >= last_text_x + 5:  # 5px отступ между метками
    painter.drawText(int(text_x), 20, time_text)
    last_text_x = text_x + text_width
```

## Преимущества

1. **Читаемость**: Метки времени никогда не накладываются друг на друга
2. **Адаптивность**: Автоматическая подстройка под любой масштаб
3. **Удобство**: Округление до "круглых" значений (5, 10, 15, 30, 60 секунд)
4. **Производительность**: Эффективный алгоритм без излишних вычислений

## Технические детали

Изменения внесены в методы `drawForeground` и `drawBackground` класса `TimelineGraphicsScene` в файле `src/views/widgets/timeline.py`.

### Для меток времени (`drawForeground`):
- Динамический расчет интервала меток времени
- Адаптивное округление до удобных значений
- Центрирование текста относительно засечки
- Проверка перекрытия меток
- Минимальное расстояние 80 пикселей между метками

### Для сетки (`drawBackground`):
- Адаптивный интервал линий сетки
- Синхронизация с интервалом меток времени
- Минимальное расстояние 60 пикселей между линиями
- Сохранение визуальной согласованности

### Общие улучшения:
- Сохранение обратной совместимости с нормальным масштабом
- Идентичные алгоритмы расчета для меток и сетки
- Эффективная производительность без излишних вычислений
