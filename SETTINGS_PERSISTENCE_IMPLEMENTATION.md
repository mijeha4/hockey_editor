# Реализация сохранения и применения настроек режима отрезка

## Описание проблемы

Ранее настройки режима отрезка (режим записи) сохранялись в `config.json` при изменении, но не применялись автоматически при перезапуске приложения. Это приводило к ситуации, когда:
- Настройки сохранялись корректно
- При перезапуске приложение использовало значения по умолчанию
- Пользователь видел несоответствие между сохраненными настройками и текущим режимом работы

## Функциональность

### При запуске приложения:
1. Автоматически загружаются настройки из `config.json`
2. Если файл не найден или поврежден - используются значения по умолчанию
3. Настройки применяются к `TimelineController`
4. Индикатор режима в статус-баре отображает текущие настройки

### При изменении настроек:
1. Настройки сохраняются в `config.json`
2. Индикатор режима немедленно обновляется
3. Изменения применяются к работающим компонентам

### Отображение в интерфейсе:
- **Фиксированная длина**: "Режим: Фиксированная длина | Длительность: 10с | Pre-roll: 3с | Post-roll: 0с"
- **Динамический**: "Режим: Динамический | Pre-roll: 3с | Post-roll: 0с"

## Преимущества реализации

1. **Полная автоматизация**: Настройки загружаются и применяются без вмешательства пользователя
2. **Надежность**: При ошибках загрузки используются безопасные значения по умолчанию
3. **Удобство**: Пользователь видит текущий режим в интерфейсе
4. **Совместимость**: Сохраняется обратная совместимость с существующими настройками
5. **Гибкость**: Все параметры (pre-roll, post-roll, fixed_duration) сохраняются и применяются

## Использование

Пользователь может:
1. Изменить режим записи в настройках
2. Настройки автоматически сохранятся
3. При перезапуске приложения настройки будут применены
4. Текущий режим отображается в статус-баре главного окна

Функциональность полностью соответствует требованиям и обеспечивает бесшовный опыт использования приложения.
